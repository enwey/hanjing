# 鼾静健康诊所 - 项目开发与设计准则

## 视觉与交互合理性要求
以后在与用户沟通或实现需求时，必须严格考查界面在**视觉呈现上的合理性**与**用户体验（UX）的直观性**：

1. **避免视觉歧义图标**：所选图标或矢量图形必须在小屏幕和紧凑布局中依然具有明确且直观的语义。禁止使用容易产生歧义的图形（例如，避免使用可能被误认为“搜索/放大镜”的斜角标签图标作为费用图标）。
2. **严禁无意义的装饰性图标**：不要为了填充空间而强行使用没有实际意义或冗余的图标或 Emoji。保持核心信息纯净、清爽。
3. **标签闭合与语法自检**：在每次修改 `WXML`、`HTML` 等模板文件时，必须严格执行标签匹配检查，严防多余闭合角括号（如 `>`）暴露在外，沦为“无意义的杂乱字符/箭头”。
4. **统一区分符设计**：支付及确认按钮中的文本与动态金额之间，保持使用一致且优雅的分隔符（例如小圆点 ` · `），确保阅读顺畅。
5. **多维度效果验证（严格执行）**：对于所有的界面修改，严禁“只修改代码而不作最终效果核对”。交付前必须在**逻辑层面**与**视觉层面（如微信开发者工具模拟器、预览效果等）**进行双重交叉验证，确保绝对无渲染丢失、遮挡、残缺或错位现象后方可交付。
