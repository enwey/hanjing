# mp-weixin-rewrite

这是与原始 `mp-weixin` 并行存在的小程序重写工程。

## 当前目标

- 不改动原始小程序代码
- 建立可读、可维护、可验证的重写结构
- 先完成应用底座、共享层、完整路由骨架
- 再按业务域逐步迁移真实页面和逻辑

## 当前完成情况

- 已同步原始小程序全部路由到新工程
- 已建立独立的 `api`、`services`、`stores`、`components`、`common` 分层
- 已完成首页、预约、睡眠评估、治疗追踪、个人中心等一级入口页的业务化重写
- 已补齐邀请好友海报生成与保存、评估与录音结果页、登录承接、部分二级业务页闭环
- 已生成并通过重写工程完整性校验脚本 `validate-rewrite.js`

## 验证方式

- 运行 `node validate-routes.js` 校验全部路由文件完整性
- 在微信开发者工具中将项目目录指向 `mp-weixin-rewrite` 进行页面验证

## 迁移原则

1. 原始项目只读，不直接修改
2. 先迁移底层模型和共享能力
3. 每个业务域完成后再进行真机/开发者工具验证
4. 只接受真实接口和真实状态，不引入假数据兜底
