# 鼾静健康诊所管理后台 —— UI设计与代码实现差异比对报告

本报告逐页比对管理后台的 UI 设计网页（`04_管理后台_一级页面.html` 与 `05_管理后台_深层页面.html`）与 Vue 代码库中实际页面实现的差异。

---

## 一、 一级主页面 (L1 Views)

### 1. 数据看板 (Dashboard)
* **代码文件**：`src/views/dashboard/index.vue`
* **比对差异**：
  * **KPI 数据卡片**：
    * **UI 设计**：固定展示 4 张卡片 —— 今日预约 (156)、本月营收 (¥38.6w)、累计患者 (2,847)、到诊率 (92.3%)。
    * **Vue 实现（增强）**：提供了“今日/本周/本月”的时间筛选按钮。点击不同时间，KPI 栏的数据和趋势（以及图表折线）会动态切换。

### 2. 预约管理 (Appointment Management)
* **代码文件**：`src/views/appointment/index.vue`
* **比对差异**：
  * **数据同步**：字段、过滤项和列表与 UI 完全一致。仅部分统计数字由于 mock 数据而有小幅正常出入。

### 3. 患者管理 (Patient Management)
* **代码文件**：`src/views/patient/index.vue`
* **比对差异**：
  * **列表列结构定义（主要差异）**：
    * **UI 设计**：合并展示 **“患者信息”** 列（包含头像、姓名、性别、年龄、手机号）。
    * **Vue 实现**：将其拆分为独立的 **“姓名”**、**“性别”**、**“年龄”**、**“手机号”** 4 列。此外还增加了一列 **“状态”**（是否启用状态）。
  * **操作按钮位置**：
    * **UI 设计**：标题行右侧放置有“📥 导出”和“➕ 手动建档”按钮。
    * **Vue 实现**：“新建患者”和“导出”按钮放置在下方的筛选栏内，标题行右侧为空。

### 4. 医生管理 (Doctor Management)
* **代码文件**：`src/views/doctor/index.vue`
* **比对差异**：
  * **页面核心结构缺失（严重差异）**：
    * **UI 设计**：页面为**双层结构**，上方展示 3 个医生名片卡片，下方直接展示一个 **“本周排班 · 5/25 - 5/31” 的排班周历表格**。
    * **Vue 实现**：当前页面**仅展示了医生卡片网格**，排班表格已被剥离至二级的排班管理子页面 `doctor/schedule.vue` 中，**主页面缺失了底部的排班表格概览**。

### 5. 门店管理 (Store Management)
* **代码文件**：`src/views/store/index.vue`
* **比对差异**：
  * **卡片指标定义（主要差异）**：
    * **UI 设计**：门店卡片底部 4 列统计数据为：**“坐诊医生”**、**“监测设备”**、**“本月预约”**、**“本月营收”**。
    * **Vue 实现**：底部 3 列统计数据改为了：**“诊室”**、**“在岗医生”**、**“日均到诊”**。**缺失了“监测设备数”与“本月营收”这两个核心运营指标**。
  * **页面样式**：UI 原型中卡片左侧包含一个精致的 `🏥` 大图标背景框，Vue 版本目前只显示纯文字详情。

### 6. 分销总览 (Distribution Overview)
* **代码文件**：`src/views/distribution/index.vue`
* **比对差异**：
  * **列表字段差异（主要差异）**：
    * **UI 设计**：列包括 推广员、等级、**“一级下线”**、**“二级下线”**、推广订单、累计佣金、**“可提现”**、状态、操作。
    * **Vue 实现**：列改为了 姓名、手机号、等级、**“团队”**、订单数、**“交易额”**、累计佣金、**“待结算”**、加入日期、来源、操作。**缺少了一级/二级下线人数的具体拆分，以及“可提现”的额度展示**。

### 7. 提现审核 (Withdrawal Audit)
* **代码文件**：`src/views/distribution/withdraw.vue`
* **比对差异**：
  * **字段缺失（主要差异）**：
    * **UI 设计**：表格中含有 **“手续费”** 和 **“到账金额”** 列（即提现扣除手续费后的实际打款金额）。
    * **Vue 实现**：仅显示了“申请金额”，**缺失了“手续费”与“到账金额”的展示与自动扣除计算**。
  * **批量操作缺失**：
    * **UI 设计**：支持多选表格行，提供 **“✅ 批量通过”** 按钮。
    * **Vue 实现**：目前仅支持单条提现申请的点击“通过/驳回”，无批量审核入口。

### 8. 订单管理 (Order Management)
* **代码文件**：`src/views/order/index.vue`
* **比对差异**：表格字段与原型对齐，无实质差异。

### 9. 系统设置 (System Settings)
* **代码文件**：`src/views/settings/index.vue`
* **比对差异**：
  * **页面架构与排版（严重差异）**：
    * **UI 设计**：页面为 **单页 2x2 看板式表单**，分成 **“📅 预约设置”**、**“💰 分销设置”**、**“🔔 通知设置”** 和 **“🔐 权限管理”** 四个并列大板块。
    * **Vue 实现**：目前被改为了 **Tab 标签切换页**（基础设置、账号管理、通知模板）。
  * **配置字段缺失（主要差异）**：
    * **预约设置**：UI 原型里的**“预约提前天数”**、**“每日可约时段”**、**“时段间隔(分钟)”**、**“取消预约截止时限”** 等配置在 Vue 代码中**被完全遗漏**。
    * **分销设置**：UI 原型里的**“一级佣金比例(%)”**、**“二级佣金比例(%)”**、**“最低提现金额(起提额)”**、**“提现手续费率(%)”**、**“是否自动审核”** 在 Vue 中**被完全遗漏**。
  * **功能重叠**：
    * Vue 将“管理员账号管理”做成了一个 Tab 嵌在设置页内，但在 router 中其实还额外保留并实现了一个独立的 L2 级页面（`settings/admin.vue`）。

### 10. 操作日志 (Operation Log)
* **代码文件**：`src/views/settings/log.vue`
* **比对差异**：列表字段与筛选参数完全对齐，无差异。

## 二、 二级/三级深层页面 (L2/L3 Views)

1. **预约详情 (L2)**：对齐，无明显差异。
2. **新建预约 (L2)**：对齐，排班选择格子逻辑一致。
3. **患者详情 (L2)**：Tab 配置与右上角操作完全一致。
4. **随访任务 (L3)**：随访类型、计划、执行与新建表单对齐。
5. **医生排班编辑 (L2)**：排班出诊及休息日修改、保存与复制排班功能对齐。
6. **门店数据报表 (L2)**：折线图和医生接诊数/营业额排行榜对齐。
7. **推广员详情 (L2)**：下级团队和佣金流水完全一致。
8. **推广商品管理 (L2)**：比例配置与上下架流程完全契合。
9. **订单详情 (L2)**：收货详情、分销分成明细对齐。
10. **退款审核 (L2)**：同意/拒绝流程与多状态筛查列表对齐。
11. **角色权限编辑 (L2)**：权限多选矩阵表格完全对齐。
12. **管理员账号管理 (L2)**：禁用与重置流程对齐。
13. **科普文章编辑 (L2)**：富文本编辑区工具栏、关联对象完全一致。
14. **轮播图管理 (L2)**：移动排序与状态控制一致。
