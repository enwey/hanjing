# 鼾静健康诊所 - 系统日志与可排查性审计报告

针对“鼾静健康诊所”微信小程序（C端）与后端服务的日志系统及可排查性（Observability & Log Analysis），我们进行了全面审计和重构加固，并于 2026 年 7 2 日完成了相关部署。以下是详细方案：

---

## 一、 系统物理日志结构分布与排查文件

后端服务已在物理磁盘上建立了 3 大核心分类日志文件，实现了对请求、异常和行为变更的归口管理：

### 1. HTTP 访问访问日志 (`requests.log`) (已重构加固)
* **原隐患**：原有的访问日志中间件仅记录请求发起时的动词和 URL，无法捕获该请求最终是成功（200）、客户端错（400）、服务端错（500）以及每个接口的处理时间。这给性能调优和故障重现带来了极大障碍。
* **重构后格式**：
  ```text
  [2026-07-02T13:09:31.423Z] 127.0.0.1 GET /api/v1/user/profile 200 - 4ms
  [2026-07-02T13:09:45.110Z] 192.168.1.100 POST /api/v1/user/upload 400 - 12ms
  ```
  * **记录参数**：`[时间戳] [客户端IP] [HTTP Method] [请求URL] [状态码] - [耗时(ms)]`。
  * **性能收益**：利用 `res.on('finish')` 事件在发包完成时统一记录，开发者可以仅凭该文件快速筛选出所有的慢查询（如耗时 > 100ms 的接口）及失败接口。

### 2. 警报与未捕获异常日志 (`alerts.log`)
* **作用**：记录系统运行期间发生的所有 `500` API 接口严重抛错、未捕获的全局进程崩溃风险异常（Uncaught Exception）以及后台常驻微服务的计算失败。
* **记录参数**：发生警报时，系统会将 `[报警时间戳] [告警等级] [错误标题]: [描述文案] \n Error: [报错简述] \n Stack: [详细调用栈 Trace]` 完整写入，极速定位问题发生的代码行号。

### 3. 用户登录专项错误日志 (`login_errors.log`)
* **作用**：针对微信小程序登录过程中三方接口返回异常、解密用户 Session 失败等场景进行专项监控，记录登录发包的 payload 和错误原因，防止登录故障发生时排查无门。

---

## 二、 日志爆盘（Disk Exhaustion）防护机制 (已落实)

为防止高频访问或遭受恶意请求攻击时日志文件不断膨胀耗尽系统磁盘，系统对日志写入实施了严格的**文件体积保护限度机制**：
```javascript
fs.stat('./requests.log', (err, stats) => {
  if (err || stats.size < 50 * 1024 * 1024) { // 限制单文件上限为 50MB
    fs.appendFile('./requests.log', logMsg, () => {});
  }
});
```
* **效果**：当单个日志文件增长至 50MB 阈值后，系统将自动锁定，只输出至标准控制台而暂停向物理文件追加，保证了磁盘存储的物理稳定性。

---

## 三、 审计结论

“鼾静健康诊所”项目具备了成熟、细致的日志体系。通过请求级细粒度访问日志、错误级调用栈告警日志和防爆盘机制的相互协作，使得系统线上运行具备极高可排查性。
