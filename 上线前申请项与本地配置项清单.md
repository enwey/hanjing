# 上线前申请项与本地配置项清单

本文档把上线前事项拆成两类：

- 需要去微信公众平台、微信支付、域名证书平台申请或配置的
- 只需要在你自己的服务器、本地部署环境里配置的

---

## 一、需要去平台申请或配置的项目

| 项目 | 去哪里处理 | 你需要拿到/完成什么 | 是否必须 | 不做会怎样 |
| --- | --- | --- | --- | --- |
| 正式小程序发布资格 | 微信公众平台 | 确认正式小程序已认证，可提审、可发布 | 是 | 无法正式上线 |
| 合法 `request` 域名 | 微信公众平台 | 配置正式接口域名，如 `https://api.hanjing.com` | 是 | 小程序请求接口会失败 |
| 合法 `uploadFile` 域名 | 微信公众平台 | 配置文件上传域名 | 是 | 病历附件、图片上传失败 |
| 合法 `webSocket` 域名 | 微信公众平台 | 配置客服 WebSocket 域名 | 是 | 在线客服无法连接 |
| 合法 `downloadFile` 域名 | 微信公众平台 | 如有附件/文件下载，配置下载域名 | 建议 | 下载类功能可能失败 |
| 隐私保护指引 | 微信公众平台 | 配置与实际能力一致的隐私说明 | 是 | 审核风险、合规风险 |
| 位置权限说明 | 微信公众平台 | 与“查找附近门店”等真实用途一致 | 是 | 审核不通过或用户授权失败 |
| 录音相关隐私说明 | 微信公众平台 | 说明鼾声录音采集用途 | 是 | 录音能力有审核/合规风险 |
| 手机号授权合规说明 | 微信公众平台 | 说明登录、建档、预约用途 | 是 | 审核或用户授权链路异常 |
| 订阅消息模板 | 微信公众平台 | 配置预约成功、提醒、复诊等模板 | 视业务而定 | 模板消息发不出去 |
| 微信支付商户号 | 微信支付商户平台 | 拿到正式商户号 | 是 | 无法正式收款 |
| 微信支付证书序列号 | 微信支付商户平台 | 拿到商户证书序列号 | 是 | 无法正常签名下单 |
| 微信支付 APIv3 密钥 | 微信支付商户平台 | 配置 APIv3 Key | 是 | 支付回调无法解密 |
| 微信支付商户私钥 | 微信支付商户平台 | 下载/生成私钥并安全保存 | 是 | 无法正常签名支付请求 |
| 支付回调公网地址 | 你的公网域名 + 微信支付后台 | 准备正式 HTTPS 回调地址 | 是 | 支付成功后订单状态无法回写 |
| HTTPS 证书 | 域名/云服务平台 | 给正式域名配置有效 HTTPS 证书 | 是 | 小程序正式请求和支付回调都不可用 |
| WebSocket 的 `wss` 证书链 | 域名/云服务平台 | 确保客服链路走 `wss://` | 是 | 小程序客服连接失败 |
| 飞书/钉钉/企微告警 webhook | 对应平台 | 申请一个机器人 webhook 地址 | 强烈建议 | 线上报错只能手工查日志 |

## 二、只需要服务器本地配置的项目

| 项目 | 文件路径或位置 | 参数名 | 你要做什么 | 是否必须 | 不做会怎样 |
| --- | --- | --- | --- | --- | --- |
| 数据库地址 | `hanjing-clinic-backend/.env` | `DB_HOST` | 改成正式数据库地址 | 是 | 后端起不来或连错库 |
| 数据库端口 | `hanjing-clinic-backend/.env` | `DB_PORT` | 改成正式端口 | 是 | 数据库连接失败 |
| 数据库账号 | `hanjing-clinic-backend/.env` | `DB_USER` | 改成正式账号 | 是 | 无法连接数据库 |
| 数据库密码 | `hanjing-clinic-backend/.env` | `DB_PASSWORD` | 改成正式密码 | 是 | 无法连接数据库 |
| 数据库名 | `hanjing-clinic-backend/.env` | `DB_NAME` | 确认正式库名 | 是 | 读写错误数据库 |
| 服务端口 | `hanjing-clinic-backend/.env` 或 PM2 | `PORT` | 与 nginx/网关保持一致 | 是 | 外网访问不到服务 |
| 运行环境 | `hanjing-clinic-backend/.env` 或 PM2 | `NODE_ENV` | 设为 `production` | 是 | 生产逻辑不生效 |
| 登录签名密钥 | `hanjing-clinic-backend/.env` | `JWT_SECRET` | 配置强随机密钥 | 是 | token 安全性不足 |
| 敏感信息加密密钥 | `hanjing-clinic-backend/.env` | `PII_ENCRYPTION_KEY` | 配置正式密钥 | 是 | 手机号/身份证加密不安全 |
| 小程序 AppID | `hanjing-clinic-backend/.env` | `WX_MINI_APP_ID` | 填正式 AppID | 是 | 微信开放接口失败 |
| 小程序 AppSecret | `hanjing-clinic-backend/.env` | `WX_MINI_APP_SECRET` | 填正式 AppSecret | 是 | 微信开放接口失败 |
| 支付 AppID | `hanjing-clinic-backend/.env` | `WECHAT_APP_ID` 或 `WX_APPID` | 填正式支付 AppID | 是 | 支付失败 |
| 商户号 | `hanjing-clinic-backend/.env` | `WECHAT_PAY_MCH_ID` | 填正式商户号 | 是 | 支付失败 |
| 证书序列号 | `hanjing-clinic-backend/.env` | `WECHAT_PAY_SERIAL_NO` | 填正式值 | 是 | 支付失败 |
| 商户私钥 | `hanjing-clinic-backend/.env` | `WECHAT_PAY_PRIVATE_KEY` | 填正式私钥 | 是 | 支付失败 |
| APIv3 密钥 | `hanjing-clinic-backend/.env` | `WECHAT_PAY_API_V3_KEY` | 填正式密钥 | 是 | 回调解密失败 |
| 通用支付回调地址 | `hanjing-clinic-backend/.env` | `WECHAT_PAY_NOTIFY_URL` | 填正式 HTTPS 地址 | 是 | 支付回调失败 |
| 预约支付回调地址 | `hanjing-clinic-backend/.env` | `WECHAT_APPOINTMENT_PAY_NOTIFY_URL` | 填正式 HTTPS 地址 | 是 | 预约支付状态无法更新 |
| 禁止前端伪确认支付 | `hanjing-clinic-backend/.env` 或 PM2 | `ALLOW_CLIENT_APPOINTMENT_PAY_CONFIRM` | 保持 `false` | 是 | 支付状态可被绕过 |
| 关闭 mock 支付 | `hanjing-clinic-backend/.env` | `ENABLE_MOCK_WECHAT_PAY` | 不开启或明确关闭 | 是 | 生产误走 mock 支付 |
| 对外基础地址 | `hanjing-clinic-backend/.env` | `PUBLIC_BASE_URL` | 填正式公网地址 | 建议 | 链接拼接异常 |
| API 对外地址 | `hanjing-clinic-backend/.env` | `API_PUBLIC_BASE_URL` | 填正式 API 地址 | 建议 | 后台某些外链异常 |
| 告警地址 | `hanjing-clinic-backend/.env` 或 PM2 | `ALERT_WEBHOOK_URL` | 填机器人 webhook | 强烈建议 | 无法自动告警 |
| PM2 生产配置 | `hanjing-clinic-backend/ecosystem.config.cjs` | `env_production.*` | 核对生产参数 | 是 | 生产参数不生效 |
| 日志目录 | 服务器文件系统 | `logs/` | 确保目录存在且可写 | 是 | PM2 日志写不进去 |
| 请求日志文件 | 服务器文件系统 | `requests.log` | 做日志轮转与磁盘监控 | 建议 | 日志持续膨胀 |
| 告警日志文件 | 服务器文件系统 | `alerts.log` | 做日志轮转与备份 | 建议 | 告警排查不方便 |
| 小程序上传配置 | `mp-weixin/project.config.json` | `urlCheck` | 建议改为 `true` | 强烈建议 | 域名问题上传前发现不了 |
| 小程序上传配置 | `mp-weixin/project.config.json` | `uploadWithSourceMap` | 建议改为 `false` | 强烈建议 | 正式包带源码映射 |

## 三、最先去做的 5 件事

1. 去微信公众平台把正式域名白名单配好
2. 去微信支付商户平台拿齐支付参数
3. 准备正式 HTTPS 域名和证书
4. 在服务器 `.env` 里补齐正式环境变量
5. 用生产配置跑通一次登录、预约、支付、上传、客服 WebSocket

---

## 四、你可以简单理解为

### 需要“去申请”的

- 微信支付
- 微信小程序后台域名白名单
- 订阅消息模板
- 隐私合规配置
- HTTPS 证书
- 告警 webhook

### 不需要“申请”，只需要你自己填

- 数据库参数
- 后端 `.env`
- JWT 密钥
- 加密密钥
- PM2 参数
- 小程序开发者工具上传配置
