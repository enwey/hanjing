## 1. 项目概述

### 1.1 产品简介

**鼾静健康诊所**是一家专注于睡眠呼吸障碍（打鼾/鼾症/OSAS）的医疗健康机构，核心产品为定制式可调舌型阻鼾器（NMPA二类医疗器械），采用无创下颌前移技术方案。

本项目为该机构的微信小程序端 + 管理后台的完整数字化系统。

### 1.2 核心功能模块

| 模块 | 说明 |
|------|------|
| **预约就诊** | 在线预约医生/门店/时段，支持改约取消，消息提醒 |
| **睡眠评估** | 埃普沃斯嗜睡量表（ESS）+ AI鼾声分析，输出健康评分 |
| **治疗追踪** | 阻鼾器佩戴记录、调整日志、治疗时间线、复诊提醒 |
| **产品中心** | 医疗器械商品展示、购买、订单管理 |
| **个人中心** | 病历档案、家庭成员、会员权益、设置 |
| **二级分销** | 推广员邀请体系、佣金结算、提现管理 |
| **管理后台** | 数据看板、预约/患者/医生/门店/订单全链路管理 |

### 1.3 目标用户

- **C端患者**：鼾症/OSAS患者及家属，30~60岁为主
- **B端推广员**：参与分销的合作伙伴/老患者
- **管理端**：诊所运营人员、医生、门店管理员、系统管理员

## 2. 技术架构

### 2.1 前端（小程序端）

```
技术栈：微信原生小程序（WXML + WXSS + JS）
         或 uni-app（Vue3 + TS，可同时编译微信/支付宝/H5）

推荐方案：uni-app + Vue3 + TypeScript
UI框架：uni-ui 或 TDesign Mobile（腾讯官方，契合设计规范）
状态管理：Pinia
网络请求：自封装 request 工具（wx.request 封装）
图表：wx-charts 或 echarts-for-weixin
图片上传：阿里云OSS / 腾讯云COS
```

### 2.2 后端

```
语言：Node.js（NestJS）或 Python（FastAPI）
数据库：MySQL 8.0（主业务） + Redis（缓存/Session/限流）
文件存储：腾讯云 COS
消息推送：微信订阅消息（subscribeMessage）
短信：腾讯云SMS
支付：微信支付 v3 API
```

### 2.3 管理后台

```
框架：Vue 3 + TypeScript + Vite
UI组件库：TDesign Web（tdesign-vue-next）
路由：Vue Router 4
状态：Pinia
图表：ECharts 5
富文本：TinyMCE 或 wangEditor
```

### 6.1 通用规范

```
Base URL: https://api.hanjing.com/v1
协议: HTTPS
格式: JSON
认证: Authorization: Bearer {access_token}
签名: 无（Token已足够）
```

**统一响应格式：**

```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "timestamp": 1749000000000
}
```

**错误码：**

| code | 说明 |
|------|------|
| 0 | 成功 |
| 1001 | 未登录 / Token失效 |
| 1002 | 权限不足 |
| 1003 | 参数错误 |
| 1004 | 资源不存在 |
| 1005 | 操作频繁，请稍后重试 |
| 2001 | 时段已被预约 |
| 2002 | 超出每日预约上限 |
| 3001 | 余额不足 |
| 3002 | 提现金额低于最低限额 |

### 6.2 认证接口

```
POST /auth/wx-login           微信登录（code换取token）
POST /auth/bind-phone         绑定手机号
POST /auth/refresh            刷新token
DELETE /auth/logout           退出登录
```

**登录流程：**
```
小程序 wx.login() → 获取 code
→ 发送 code 至后端
→ 后端换取 openid/session_key
→ 生成 access_token（JWT，7天有效）
→ 返回用户信息 + token
```

## 7. 核心业务流程

### 7.1 预约就诊完整流程

```
[用户端]
首页 → 点击"立即预约"
↓
门店选择 → 地图/列表展示门店，选择目标门店
↓
医生详情 → 查看医生信息、评分、可约时段
↓
选择时段 → 日历选日期 → 格子选时段（绿=可约/灰=已满）
↓
预约确认 → 填写症状描述，核对信息
↓
支付挂号费（如需）→ 微信支付
↓
预约成功 → 推送订阅消息提醒

[后端处理]
1. 校验时段库存（Redis锁防并发超卖）
2. 创建预约记录
3. 扣减时段库存
4. 推送确认消息（订阅消息）
5. 写入分销来源记录

[就诊当天]
系统自动发送就诊提醒（预约前1小时）
患者到店签到（门诊台扫码 or 管理后台操作）
医生就诊完成 → 管理后台标记完成 → 创建治疗记录

[就诊后]
系统14天后推送随访消息
管理员创建随访任务
```

### 7.2 ESS量表评估流程

```
进入睡眠评估 → 点击"开始量表"
↓
8道题依次作答（0-3分，情境题）
题目：在特定情境下打盹/睡着的可能性
（乘车/阅读/开会/看电视/躺下休息/坐着与人交谈/饭后休息/堵车等）
↓
计算总分（0-24）：
  0-7：正常
  8-9：轻度嗜睡
  10-15：中度嗜睡
  16-24：重度嗜睡
↓
输出评估报告（评分环形图 + 建议 + 推荐就诊）
```

### 7.3 AI鼾声分析流程

```
点击"AI鼾声分析" → 引导用户开启麦克风权限
↓
录制1-3分钟睡眠鼾声（实时波形可视化）
↓
上传至OSS → 后端调用音频分析服务
↓
分析指标：
  - 平均/峰值分贝
  - 鼾声占比（占录音时长的百分比）
  - 疑似呼吸暂停事件次数
  - 风险等级（低/中/高）
↓
生成报告 → 推荐预约就诊
```

### 7.4 阻鼾器治疗追踪流程

```
就诊后，医生在后台创建治疗记录（器械型号/前移量/医嘱）
↓
患者小程序端可见治疗方案
↓
每日佩戴打卡（记录时长+舒适度评分）
↓
治疗时间线展示（阶段标记：初始/调整/稳定）
↓
到达下次调整日期时，推送提醒预约复诊
↓
复诊后更新调整记录，进入下一个治疗周期
```

## 8. 分销系统规范

### 8.1 推广员等级体系

| 等级 | 图标 | 晋升条件 | 一级佣金 | 二级佣金 |
|------|------|---------|---------|---------|
| 银牌 | 🥈 | 注册即可申请 | 10% | 3% |
| 金牌 | 🥇 | 累计带单 ≥ 10 单 | 15% | 5% |
| 钻石 | 💎 | 累计带单 ≥ 50 单 | 20% | 8% |

> **注**：UI设计稿中佣金率为金牌标准（15%+5%），实际以后台配置为准。

### 8.2 佣金计算规则

```
一级佣金 = 订单实付金额 × 商品佣金率（默认按推广员等级）
二级佣金 = 订单实付金额 × 商品二级佣金率

佣金状态流转：
  下单 → 【冻结】→ 订单完成后 14 天 → 【已结算/可提现】
  退款/撤单 → 【已撤销】（已结算佣金需冻结等额余额冲抵）
```

### 8.3 提现规则

```
最低提现金额：50元
提现手续费：微信零钱 0%，银行卡 1%（最低1元）
审核时效：工作日 1 个工作日内审核
到账时效：审核通过后 2 个工作日内打款
```

### 8.4 推广商品管理

- 每个商品单独配置推广佣金率
- 支持上架/下架控制推广范围
- 推广员可查看推广素材（文案/图片/海报/短视频/朋友圈文案/话术）

### 8.5 关系绑定规则

```
用户A扫推广员B的二维码进入小程序 → 绑定B为A的一级推广员
推广员B的上级是推广员C → C为A的二级推广员关系

绑定规则：
1. 一经绑定，永久有效（不可更换）
2. 推广员本人下单不产生自身佣金
3. 同一用户只绑定第一次来源
```

### A. 开发优先级建议

- 科普内容
- 随访管理

### B. 安全要求

- 所有接口 HTTPS
- 身份证号、银行卡号加密存储（AES-256）
- 银行卡号前端展示仅显示后4位
- 提现操作需二次验证（微信授权确认）
- 管理后台操作全量日志
- 敏感接口（提现审核/系统设置）需RBAC验证
- 微信支付回调验签

### C. 性能要求

| 指标 | 目标值 |
|------|--------|
| 页面首屏时间 | < 1.5s |
